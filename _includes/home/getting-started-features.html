Lore is designed/built to be flexible. It follows a pattern of cascading overwrites. You set patterns for the project, and then override them as you need.

Lore is designed/built to be extensible. Lore is implemented as a series of plugins, and all of it's behaviors are exposed through configuration files. The framework is implemented as a series of plugins, And if that's not enough, you can even write your own plugins to override or extend the frameworks engine. Lore has opinions, but it also respects yours.

Lore is designed/built to be scalable. If you're an experienced developer, then you know how difficult it can be to maintain agility while supoorting an evolving application.
