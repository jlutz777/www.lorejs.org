Purpose

While Lore's goal is to hide as much of the complexity and learning curve associated with building React applications as possible (at least initially), it's ability to do so becomes more difficult as your application becomes more complex and needs to be tailored to your specific situation.

At some point, your application will no longer fit as neatly within the default conventions provided by the framework, and at that point you'll need to learn more about these concepts to help the framework adapt to your specific situation.

The topics in this section make up the fundamental concepts and principles of Lore and React development in general. Lore's goal is to put off requiring you to learn them for as long as possible until the specific needs of your application require you to do so.

Mastering these concepts will remove any trace of sense of "magic" that Lore provides, as Lore itself is really just a set of conventions and sensible defaults connecting the underlying libraries together.

What libraries is Lore built on?

Lore is built on a set of libraries that are very popular within the React community.

  • React - view tier
  • Webpack - build system
  • Redux - architecture
  • React-Router - routing
  • Gulp - task runner
