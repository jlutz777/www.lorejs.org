Visual Cues

Useful for providing the user with a visual indication that some action is being performed but has no yet completed. Examples include fetching, updating and creating data. Doing this well enables your components to be completely data-driven.

Visualization

This video demonstrates what visual cues look like. Screenshots are from the Simply Social prototype that Invision provides you when you sign up for an account.

Architecture Challenge

The core challenge is making sure that any information that components need to know in order to render the correct view is contained within the data they're provided.

This video describes how Lore implements support for visual cues.
