Screenshots are from the _Simply Social_ prototype that [Invision](https://www.invisionapp.com/) provides you when you sign up for an account.
### Usage For a demonstration of pagination in action, please see the [pagination example](https://github.com/lore/lore/tree/master/examples/pagination). Pagination is very similar to filtering, except that you include a second property in the `getState` call named `pagination`. The properties passed to `where` and `pagination` are then combined to form the query parameters for the final URL for the API call. For the example code below, the final URL might look like this: ``` https://api.myapp.com/posts?authorId=123&page=2 ``` When implementing pagination, it's recommended to use the route specify which page of data to display, such as: ``` https://www.myapp.com/posts?page=2 ``` This will make it easy to support as everytime the route changes the `connect` call in the components will get a change to rebuild their data request for the new page of data (if it's changed).
