## Written Steps ### The Config Folder Before we can fetch data from our API server we need to tell Lore where our API server is located. At the root of your project you should see a folder called `config`. This folder exposes all of the toggles and knobs you can use to tailor Lore's behavior to your specific needs. Navigate into this folder and open a file called `models.js`. Toward the top of this file you should see a commented out line that looks like this: ```js module.exports = { // apiRoot: 'https://api.example.com', } ``` You'll see commented out sections like this in all of the config files, and they represent the built-in defaults that Lore uses. In this case, Lore assumes you're API is located at `https://api.example.com` which is obviously not going to be true. So let's tell Lore where our API is located by uncommenting out this line and changing it to the following: ```js module.exports = { apiRoot: 'http://localhost:1337', } ``` With this change in place, we're almost ready to fetch data. But first, we need to talk about what `models` are. > Aside: Environment-specific Configuration > While we won't cover it in this tutorial, you'll notice there is a `/env` folder in `/config` that contains files > called `development.js` and `production.js`. In Lore, there is a concept called a `model` that understands how to interact with this endpoint. For example, in this tutorial we'll soon be creating a `tweet` model, and this model will know how to fetch, create, update and delete tweets. But the endpoints within an API often share a lot of common conventions, and having to redefine these conventions (like the root URL for the API server) would be a lot of duplication of information. This config file exists as a way to define that common behavior in a single location, and all models in the application will inherit those properties. ### Visual Check-in If everything went well, your application should now look like this.
