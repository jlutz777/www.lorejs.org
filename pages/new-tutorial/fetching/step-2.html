## Written Steps ### Create the Tweet Model To create our tweet model run the following command: ``` lore generate model tweet ``` This command will create a model called `tweet` located at `src/models/tweet.js`. And if you open that file up, you should see a lot of comments, which show some of the options that exist to tailor behavior when interacting with the `/tweets` endpoint. But aside from the comments, this file is essentially empty: ```js module.exports = { }; ``` ### Empty File, Lots of Functionality This file may be empty, but believe it or not, the application now has the ability to fetch, create, update and delete tweets. To prove this, refresh your browser and open up the developer tools. Navigate to the console and enter this command: ``` lore.actions.tweet.find() ``` Once you do this you should see a GET request sent to `http://localhost:1337/tweets`, and the API server will respond with the first page of tweets. So basically, just by creating a model named `tweet`, you have a method you can invoke to fetch tweets. That's pretty cool! Let's put it to > If you see an error in the console like `Cannot read property 'toString' of undefined` don't worry. While the > error message itself is unhelpful, we'll be solving the issue soon (and in the future the error message will be > more clear about exactly why it appears). As for the 401 response, our request was rejected by the API server because only authorized users have permission to create tweets.
