If you look at the data for our tweet, you'll notice it includes a property called `userId` that has a number for a value. This number represents the id of the user the tweet belongs to. Taking a look at the API endpoint for `/users` we can see that a user resource looks like this: ```js { id: 1, nickname: "lucca", avatar: "https://cloud.githubusercontent.com/assets/2637399/19027072/a36f0c7a-88e1-11e6-931e-7f67fe01367b.png", createdAt: "2016-10-25T04:24:46.207Z", updatedAt: "2016-10-25T04:24:46.207Z" } ``` So just like we did with the mock tweet, we're going to create a mock user.
