To explain what this method does, open up `index.html` and find the element in the body with the id of `dialog`: ```html
...
... ``` When your application runs in the browser, Lore mounts it to the `root` element. The `dialog` element is the default target for mounting dialogs, and the `lore.dialog.show(...)` method is a helper that renders a React component to that element. > Why have two separate mounting targets? > > Lore renders dialogs outside the main application in order to prevent them from being nested inside other DOM > elements. If you nest dialogs inside other components those components can unintentionally affect the styling of > your dialogs (through classes applied to parent elements) or the behavior (such as parent components cancelling the > click events in your dialogs). ### Mounting a Component To demonstrate this utility, replace the `onClick` behavior of our `CreateButton` component with this code: ```jsx ... lore.dialog.show(function() { return (
