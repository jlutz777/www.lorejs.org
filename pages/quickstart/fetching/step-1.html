--- layout: quickstart permalink: /quickstart/fetching/step-1/ --- {% capture markdown %} # Step 1: Set the API Location In this step we're going to set the location of the API server. ### The Config Folder At the root of your project you should see a folder called `config`. This folder exposes all of the controls you can use to tailor Lore's behavior to your applications needs. Navigate into this folder and open a file called `connections.js`. Toward the top of this file you will see a commented out line that looks like this: ```js ... // apiRoot: 'https://api.example.com', ... ``` You'll see commented out sections like this in all of the config files, and they represent the built-in defaults that Lore uses. In this case, Lore assumes your API is located at `https://api.example.com`, which is obviously not true. Tell Lore where the API is located by uncommenting that line and change it to the following: ```js ... apiRoot: 'http://localhost:1337', ... ``` With this change in place, we're almost ready to fetch data. > *Environment-specific Configurations* > > You may notice a `/env` folder in `/config` that contains files called `development.js` and `production.js`.
