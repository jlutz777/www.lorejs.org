--- layout: quickstart permalink: /quickstart/fetching/step-6/ --- {% capture markdown %} # Step 6: Display Loading Message In this step we're going to use the `state` property on the data structure to display a loading message while the tweets are being fetching. ### Add a Loading Experience While not obvious right now, there is a moment before the list of tweets renders when the page simple says "Feed" with no data. The reason for that is that the data is currently being fetched, and we can't render what we don't have. Showing a blank view is a bad user experience, so let's take advantage of the `state` property on our `tweets` data to discover when the data is being fetched and render the text "Loading..." to the screen. There's a folder in Lore located at `src/constants` that contains a file called `PayloadStates`. This file contains all of the states that data can be in by default (generic states the framework will apply automatically).
