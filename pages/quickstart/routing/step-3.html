## Written Steps ### Create the Feed Component First we need to create our Feed component. Once again, we're going to use the CLI to generate that component for us: ``` lore generate component Feed ``` Next modify the `render` method so it displays some text when it renders (otherwise we won't know when it's being displayed). ); } }); ``` ### Add Feed to Routes.js Next open `routes.js` and import the Feed component. Then create a child route under `/` (the Layout). But unlike the current routes, we're going to be using the `IndexRoute` type and not the standard `Route` type. The `IndexRoute` is used when you want to say "render this component when there is no nested route". In other words, we want to display the Feed when the route is `/`, but in the tutorial we'll want to add a MyTweets component that will be display the users tweets when the route is `/users/123`. So the `IndexRoute` is used to say "render the Feed only when the route is /". Here are the changes you need to make to `routes.js`: ```jsx ... var Feed = require('./src/components/Feed'); module.exports = ( ); ``` ### Render Route Children in Layout If you refresh the application right now, you'll notice the Feed component isn't being displayed. Why is that? Well, simply put, because we haven't told the Layout where to render it! So let's do that. Open `Layout` and look in the render method for the code that looks like this: ```js
{/* Feed goes here */}
``` Modify that code to look like this: ```js
{React.cloneElement(this.props.children)}
``` When we added `Feed` to `routes.js` we listed it as a child of `Layout`. This code we just pasted says "clone my children and render them here". With this change in place, refresh the application and you should now see "Feed" displayed in the middle of the page. ### Visual Check-in If everything went well, your application should now look like this. ## Code Changes Below is a list of files modified during this step, as well as a visual diff to show you what was added or removed between this step and the last one. ### routes.js {% tabs tab1={"name": "Diff", "id": 1} | tab2={"name": "Source", "id": 2} %} {% tab id=1, is_active=true %} ![New Lore App](/assets/images/quickstart/diffs/routing/step-3-routes.png) {% endtab %} {% tab id=2 %} ```jsx var React = require('react'); var Route = require('react-router').Route; var IndexRoute = require('react-router').IndexRoute; /** * Routes are used to declare your view hierarchy * See: https://github.com/ReactTraining/react-router/blob/master/docs/API.md */ var Master = require('./src/components/Master'); var Layout = require('./src/components/Layout'); var Feed = require('./src/components/Feed'); module.exports = ( ); ``` {% endtab %} {% endtabs %} ### src/components/Feed.js {% tabs tab1={"name": "Diff", "id": 3} | tab2={"name": "Source", "id": 4} %} {% tab id=3, is_active=true %} ![New Lore App](/assets/images/quickstart/diffs/routing/step-3-feed.png) {% endtab %} {% tab id=4 %} ```js var React = require('react'); module.exports = React.createClass({ displayName: 'Feed', getStyles: function() { return { title: { textAlign: 'center' }, tweets: { marginTop: '32px' } } }, render: function() { var styles = this.getStyles(); return (
